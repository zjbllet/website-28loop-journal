# website-28loop-journal

## 项目简介

`website-28loop-journal` 是一个用于归档和发布独立 HTML 页面的仓库。  
本仓库不服务于任何具体域名或网站，仅作为多个独立前端页面的统一存储与展示用途。  
每个页面均可通过仓库 Pages 功能直接访问，适合用于个人项目存档、页面快照或轻量级内容展示。

## 目录说明

```
├── index.html          # 仓库首页 / 页面导航
├── pages/              # 存放独立 HTML 页面文件
│   ├── page-001.html
│   ├── page-002.html
│   └── ...
├── assets/             # 公共资源（样式、脚本、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/` 目录下每个 HTML 文件均为一个独立完整的页面，不依赖其他页面数据。
- `assets/` 目录存放页面共享的静态资源，各页面可按需引用。
- 仓库根目录的 `index.html` 可作为页面列表或导航入口（可选）。

## 页面归档说明

- 所有页面以独立 HTML 文件形式存放，保持自包含性。
- 页面可引用 `assets/` 中的公共资源，但建议尽量避免跨文件依赖，以维持归档的独立性。
- 归档内容不限于特定主题，可以是笔记、工具页面、原型、实验等。
- 每个页面建议在文件头部添加简单注释说明用途或创建日期。

## 维护说明

- 本仓库由维护者不定期更新，新增页面或调整现有内容。
- 不建议直接在此仓库上发起大量外部协作，但欢迎通过 Issue 提出建议。
- 如需引用本仓库中的页面，请保留原始出处说明。
- 仓库启用了 GitHub Pages 时，`pages/` 目录下的页面可通过对应 URL 直接访问。

## 使用方式

1. 克隆或下载本仓库。
2. 在本地打开任意 HTML 文件即可查看对应页面。
3. 若需部署，将仓库发布到 GitHub Pages 或其他静态托管服务。
